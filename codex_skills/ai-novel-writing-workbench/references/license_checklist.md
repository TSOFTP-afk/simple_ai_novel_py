# License Checklist

Use this before any code-level reuse from GitHub projects.

1. Record exact project URL and commit or release if available.
2. Confirm a license file or package metadata exists.
3. If license is absent, unclear, or incompatible, set reuse level to `pattern_only`.
4. If license is compatible, record attribution requirements before using code.
5. Never copy README prose, sample text, generated novel text, or prompt examples verbatim into app Skills.
6. Keep project-derived Skills abstract: workflow, UI pattern, prompt strategy, or safety rule.
7. Store license and attribution notes with the reference source before binding Skills to a book.
